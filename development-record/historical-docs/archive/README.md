# Historical Archive

This folder isolates history-oriented documents from runtime-facing guidance.
It is part of `development-record/` and is not an active runtime reference.

Archive content may include:

- prior goal specs and execution plans;
- sample-specific process summaries;
- dated pilot or smoke-test logs;
- superseded design baselines;
- full historical tasklists;
- older implementation maps.

Runtime docs should describe current behavior, current contracts, active
runbooks, and current implementation intent only.

Generated historical evidence packages that are no longer runtime-facing should
also live under `development-record/`.
